Spark & Flink 과제

GitHub → github.com/qkrwnsqja8996/YBIGTA_SPFL_hw

Flink(스트리밍)와 Spark(배치)로 같은 데이터를 처리할 때 어떤 차이가 있는지 직접 확인하는 과제입니다.
설정 파일 2개를 채우는 게 전부입니다. 값을 찾는 게 아니라, 왜 그 값인지를 이해하는 게 목표입니다.

빠른 시작

# 이미지 빌드 (최초 1회)
./scripts/cluster.sh build

# 클러스터 시작
./scripts/cluster.sh init

# 전체 시나리오 실행
./run_all.sh

# 결과 확인
cat results/result.json

준비물

  • Docker Desktop — Settings → Resources → Memory를 6GB 이상으로 올려주세요
  • Docker Compose v2
  • 디스크 10GB 이상

해야 할 것

student/ 안에서만 수정하면 됩니다.

파일내용
student/conf/flink-conf.yaml 체크포인팅 설정(간격, 모드, 저장 경로)과 병렬성 설정 — # TODO 블록을 채우세요
student/conf/spark-defaults.conf shuffle partition 수 조정 — 지금 200으로 되어 있는데, 이 상태로 실행하면 시나리오 04가 실패합니다

힌트는 각 파일 주석과 student/README.md에 있습니다. 수정 금지: docker-compose.yml, jobs/, scenarios/, scripts/

시나리오

#이름내용
01 flink-basic Flink 실시간 처리 — 설정이 맞아야 잡이 뜹니다
02 spark-batch Spark 배치 집계 — CSV 읽고 groupBy
03 stream-vs-batch 같은 데이터를 두 엔진으로 처리, latency 비교
04 spark-shuffle 파티션 수 과다 → 오버헤드 폭증. 이해하고 고치세요
05 pipeline Kafka → Flink → Spark 전체 파이프라인

5개 모두 통과하면 됩니다.

채점

항목기준
시나리오 통과5개 전부 passed: true
출력 정합성각 시나리오의 output sha256 일치

5개 전부 통과하면 PASS, 그 이상은 리더보드 순위용입니다.

리더보드

  • 1순위: scenarios_passed 많을수록
  • 2순위: penalty_ms 적을수록 (통과한 시나리오 실행 시간 합)

디렉터리 구조

YBIGTA_SPFL_hw/ ├── docker-compose.yml # 클러스터 구성 — 수정 금지 ├── run_all.sh # 전체 실행 ├── scripts/ # 클러스터 제어 — 수정 금지 ├── scenarios/ # 검증 스크립트 — 수정 금지 ├── jobs/ # Flink / Spark 잡 — 수정 금지 ├── student/ # ★ 여기서만 작업하세요 ★ │ ├── conf/ │ │ ├── flink-conf.yaml │ │ └── spark-defaults.conf │ └── README.md ├── results/ # result.json 생성 위치 └── submissions/ # 제출 파일 위치

제출 방법

./scripts/prepare_submission.sh <이름>
git commit -m "submit: <이름>"
git push

submissions/<이름>/ 안에 아래 파일만 있으면 됩니다.

submissions/
└── <이름>/
    ├── result.json
    └── conf/
        ├── flink-conf.yaml
        └── spark-defaults.conf

PR을 올리면 GitHub Actions가 자동으로 검증합니다. student/, results/ 등 다른 파일이 같이 올라오면 CI가 실패합니다. PR은 검증 통과 후 merge됩니다.
